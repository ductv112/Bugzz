**If build fails — apply these diagnostic filters (in order):**

1. **"failed to find target 'android-35'"** → API 35 platform not installed. Edit `app/build.gradle.kts` line `compileSdk = 35` → `compileSdk = 36`. The `android.suppressUnsupportedCompileSdk=36` flag in `gradle.properties` (pre-armed by Plan 01) will suppress the version warning. Re-run Step 2. Document the fallback in SUMMARY. Revert to 35 after user confirms API 35 install is complete.

2. **"Incompatible classes found" / KSP errors** → Gotcha G-1 (Kotlin/KSP mismatch). Verify `gradle/libs.versions.toml` has `kotlin = "2.1.21"` and `ksp = "2.1.21-1.0.32"` EXACTLY. If KSP 1.0.32 is not published, try 1.0.33 or 1.0.34 (same Kotlin version).

3. **"Hilt Android entry point is applied on an activity of an application not annotated with @HiltAndroidApp"** → Gotcha G-7. Verify `BugzzApplication.kt` has `@HiltAndroidApp` AND `AndroidManifest.xml` has `android:name=".BugzzApplication"`.

4. **"Could not resolve androidx.test.ext:junit:1.3.0"** → Research Assumption A3: this version may not exist yet. Edit `gradle/libs.versions.toml`: `androidxTestExtJunit = "1.2.1"`. Also `espressoCore = "3.6.1"` if needed. Re-run.

5. **"Configuration cache problem"** → Gotcha G-8. Edit `gradle.properties`: change `org.gradle.configuration-cache=true` → `org.gradle.configuration-cache=false`. Re-run.

6. **"org.gradle.java.home is invalid"** → Gotcha G-4. Verify `gradle.properties` uses `\\` (doubled) backslashes: `org.gradle.java.home=C:\\Program Files\\Android\\Android Studio\\jbr`.

7. **"Unable to find method 'composable<T>'" / "Serializer for class 'X' not found"** → Gotcha G-6. Verify every route in `Routes.kt` is annotated `@Serializable` and the `kotlin-serialization` plugin is applied in `app/build.gradle.kts`.

Document any fallback taken in the SUMMARY. Do NOT proceed to Task 2 until APK exists.

# Phase 1 Handoff — FND-08 Device Verification Runbook

**Prerequisites (user-side):**
1. Physical Android 9+ device with USB debugging enabled.
   - Settings → About phone → tap Build number 7 times (enables Developer Options)
   - Settings → Developer options → USB debugging: ON
2. USB cable connected between phone and this PC.
3. When prompted on the phone, tap "Allow" for the RSA fingerprint dialog.

**What you are verifying:**
- FND-01: `./gradlew :app:assembleDebug` produced an APK (already done in Task 1 of this plan).
- FND-08: That APK installs on your physical device and the app opens without crashing.
- FND-06: The CAMERA permission flow works as designed (first entry prompt → denial rationale → Open Settings CTA).

***

## Step 1 — Confirm the APK exists

```bash
ls -la app/build/outputs/apk/debug/app-debug.apk
```

## Step 2 — Confirm the phone is detected

```bash
"C:/Users/Admin/AppData/Local/Android/Sdk/platform-tools/adb.exe" devices
```

Expected output:
```

<your-device-serial>    device
```

If no devices are listed, verify USB cable is "data + charging" (not charge-only) and try another USB port.

***

```
Success
```
