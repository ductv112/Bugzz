## Design System

| Property | Value |
|----------|-------|
| Tool | none (Android Compose Material3 — not a web/shadcn project) |
| Preset | not applicable |
| Component library | Jetpack Compose Material3 (BOM 2026.03.00) — existing from Phases 1-3 |
| Icon library | Material Icons core (`androidx.compose.material:material-icons-core`) — no extended pack; text-label fallback if icon missing (Phase 2 D-24 pattern) |
| Font | System default (Roboto via Material3 theme) — branding font deferred to Phase 6 UX-01 |

**Design system state:** No `Theme.kt` / `Color.kt` / `Type.kt` files exist in codebase. `themes.xml` uses `android:Theme.Material.Light.NoActionBar` with black status bar. ## Spacing Scale

Declared values (multiples of 4dp — Android dp units throughout):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline text padding, border widths |
| sm | 8dp | Horizontal spacing between picker thumbnails; compact element margins |
| md | 16dp | Default section padding; button internal padding; flip button edge offset |
| lg | 24dp | Shutter bottom padding (Phase 3 D-13 carry); section vertical gaps on HomeScreen |
| xl | 32dp | HomeScreen center-stack vertical spacing between buttons (D-19) |
| 2xl | 48dp | Large layout breathing room (not used Phase 4 directly) |
| 3xl | 64dp | Page-level (not used Phase 4) |

Exceptions:
- Picker strip top padding: 16dp, bottom padding: 12dp (yields 100dp strip height with 72dp thumbnails — locked D-15)
- Touch target minimum: 44dp × 44dp on all interactive elements (WCAG 2.5.5 AA minimum for personal-use app)
- Shutter button: 72dp diameter — exceeds 44dp minimum; no exception needed
- Settings gear IconButton: 48dp × 48dp touch target (24dp icon inside 48dp box with 12dp padding each side)
- My Collection button: 160dp × 56dp (D-19 locked)

## Color

Project has no custom color palette yet. Phase 4 defines a minimal functional palette aligned with the "bug prank app" tone: dark camera viewfinder background, semi-transparent dark picker overlay, white interactive elements on camera, standard Material3 for Home screen.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FF121212` — near-black | Camera screen background (`Color.Black` already in CameraScreen); picker strip surface; Home screen background |
| Secondary (30%) | `#FF1E1E1E` — dark gray | Picker strip background overlay; disabled button surface; "Not yet available" stub card background |
| Accent (10%) | `#FFFFFFFF` — white | Shutter button fill; selected thumbnail border ring; primary "Face Filter" button; active interactive elements on camera |
| Destructive | `#FFB00020` — Material3 default error red | Error Toasts only; not used Phase 4 for any UI element (no destructive actions in this phase) |
| Surface / Home | Material3 `MaterialTheme.colorScheme.surface` | HomeScreen Scaffold background (defaults to near-white in light theme or dark in dark theme — system-controlled) |
| On-surface text / Home | Material3 `MaterialTheme.colorScheme.onSurface` | HomeScreen text, button labels |

Accent reserved for:
1. Shutter button fill (white circle)
2. Selected picker thumbnail border ring (2dp white border)
3. Primary "Face Filter" button fill on HomeScreen (uses Material3 `ButtonDefaults.buttonColors()` — system accent)
4. Filter-name text on picker (white `#FFFFFFFF` against dark picker strip background)

Color rules:
- Camera viewfinder background remains `Color.Black` (existing Phase 3 code)
- Picker strip background: `Color(0xCC1E1E1E)` — 80% alpha dark gray overlay (allows faint camera preview to bleed through bottom edge without full opacity wall)
- Disabled button text/icon: `Color.White.copy(alpha = 0.38f)` (Material3 disabled alpha convention)
- Disabled button container: `Color.White.copy(alpha = 0.12f)` overlaid on dark surface
- All camera-screen UI elements (flip, picker, shutter) use white/light colors against the dark camera preview

### 2. CameraScreen — Picker Strip Addition (D-15, D-16, D-18)

**Picker strip container:**
- `Box(modifier = Modifier.align(Alignment.BottomCenter).padding(bottom = 124dp))` — positions strip 100dp above shutter (shutter 72dp + 24dp bottom padding + 28dp gap ≈ 124dp bottom offset; exact value = 24dp shutter-bottom-padding + 72dp shutter-height + 28dp gap between shutter top and strip bottom = 124dp)
- Strip surface: `Surface(color = Color(0xCC1E1E1E), modifier = Modifier.fillMaxWidth().height(100.dp))`
- Internal: `LazyRow(state = listState, contentPadding = PaddingValues(horizontal=8.dp), horizontalArrangement = Arrangement.spacedBy(8.dp), modifier = Modifier.fillMaxSize())`

**FilterThumbnail composable (per item in LazyRow):**

Each item is a `Box(modifier = Modifier.size(88.dp).padding(vertical=8.dp))` — outer 88dp to give tap target room; inner clip to 72dp.

Inner thumbnail:
```
Box(
  modifier = Modifier
    .size(72.dp)
    .clip(RoundedCornerShape(12.dp))
    .graphicsLayer { scaleX = scale; scaleY = scale }  // animateFloatAsState
    .border(
      width = if (selected) 2.dp else 0.dp,
      color = if (selected) Color.White else Color.Transparent,
      shape = RoundedCornerShape(12.dp)
    )
    .clickable { onSelect(filterId) }
    .semantics { role = Role.Button; contentDescription = "$displayName filter" }
)
```

Inside: `AsyncImage(model = "file:///android_asset/sprites/$assetDir/frame_00.png", contentScale = ContentScale.Crop, modifier = Modifier.fillMaxSize())` via Coil 2.7

Below thumbnail: `Text(text = displayName, fontSize = 10.sp, fontWeight = FontWeight.Medium, color = Color.White, maxLines = 1, overflow = TextOverflow.Ellipsis, textAlign = TextAlign.Center, modifier = Modifier.width(72.dp).padding(top=2.dp))`

Minimum touch target for the clickable area: the full 88dp slot (72dp thumbnail + 8dp vertical padding each side = 88dp tap region) — exceeds 44dp minimum.

**Selected-state animation (D-16):**
- `val scale by animateFloatAsState(targetValue = if (selected) 1.15f else 1.0f, animationSpec = tween(durationMillis = 200, easing = FastOutSlowInEasing), label = "filterScale")`
- On tap: `scope.launch { listState.animateScrollToItem(index = selectedIndex, scrollOffset = -(viewportWidth / 2 - 44))  }` — centers selected item (44dp = half of 88dp slot)
- Scale easing: `FastOutSlowInEasing` (Material3 standard decelerate easing — snappy snap-in)
- Duration: 200ms (locked D-16)

**Picker background:** semi-transparent, does NOT intercept camera preview touch events. Picker is strictly bottom-strip; camera preview above it remains interactive (flip button, future gestures).

**Cycle Filter button removal (D-18):**
- Phase 3 `Cycle` OutlinedButton at `BottomEnd` is removed in Phase 4
- TEST RECORD button at `BottomStart` (BuildConfig.DEBUG) is retained as-is

## Implementation Notes for Executor

1. **Picker strip z-order:** The picker strip is a Compose overlay inside the `Box` that also contains `CameraXViewfinder`. Draw order in Compose Box: Viewfinder first (bottom), then picker strip, then shutter button, then flip button, then capture flash (topmost). Ensure `AnimatedVisibility` for capture flash is the last element in the `Box` content lambda.

2. **Picker bottom offset calculation:** Shutter is `Alignment.BottomCenter` with `padding(bottom=24dp)` and `size(72dp)`. The top of the shutter button is at `24dp + 72dp = 96dp` from bottom. Picker strip `height=100dp`. Gap between strip bottom and shutter top: recommend 8dp. Therefore picker strip `padding(bottom = 96dp + 8dp) = padding(bottom = 104dp)`. Verify on device — adjust if visual crowding evident.

3. **Cycle Filter button removal:** Delete the `if (BuildConfig.DEBUG) { OutlinedButton ... Text("Cycle") }` block from CameraScreen. Do NOT delete the `vm.onCycleFilter()` method yet — it may be referenced in tests. Mark as `@Deprecated` and leave empty body, or remove if tests confirm no coverage.

4. **Coil `file:///android_asset/` URI:** Coil 2.7 supports this URI scheme for asset loading via its built-in `AssetUriFetcher`. No custom fetcher needed. Test on device — if loading fails silently, check that `coil-compose` is the full artifact (not `coil-base` which may omit asset fetcher).

5. **LazyListState key:** Use `rememberLazyListState()` (not `rememberSaveable`) — portrait-lock means configuration changes are suppressed (D-21), so saveable key overhead is unnecessary.

6. **DataStore write debounce:** No debounce per D-25 discretion note. DataStore handles write coalescing internally. Rapid filter taps write each selected id; only the final settled id matters for next-launch restore.

7. **HomeScreen ViewModel:** A `HomeViewModel` is likely unnecessary for Phase 4 — HomeScreen only needs nav callbacks and Toast context. Implement as a stateless composable receiving `onFaceFilter`, `onCollection` lambdas + `LocalContext` for Toast. If state is needed later (e.g., showing a badge on My Collection), add ViewModel then.
