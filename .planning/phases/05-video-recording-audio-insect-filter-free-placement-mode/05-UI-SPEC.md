## Component Specs

### 1. Record Button — Production (D-07)

**Replaces:** Phase 2/4 TEST RECORD debug button. No `BuildConfig.DEBUG` gate — production always-visible.

**Position:** `Alignment.BottomStart` inside the camera preview `Box`, `padding(start = 24.dp, bottom = 24.dp)`. Vertically aligned with shutter button bottom edge (both share the `bottom = 24dp` offset — same lg spacing token).

**Size:** 56dp × 56dp circular Surface.

**Layout structure:**
```
Box(
  modifier = Modifier
    .size(56.dp)
    .clip(CircleShape)
    .background(Color(0xFFE53935))  // accent-record — same in both states
    .border(width = 2.dp, color = Color.White, shape = CircleShape)
    .clickable(enabled = !isRecording || recordingState is RecordingState.Active) {
        onRecordTapped()
    }
    .semantics {
        role = Role.Button
        contentDescription = if (isRecording) "Stop recording" else "Start recording"
    }
) {
    // Idle state: no inner composable — red circle with white border is the full visual
    // Recording state: white square centered inside
    if (isRecording) {
        Box(
            modifier = Modifier
                .size(20.dp)         // stop-square: 20dp inside 56dp circle
                .clip(RoundedCornerShape(4.dp))   // slight rounding on stop square (universal convention)
                .background(Color.White)
                .align(Alignment.Center)
        )
    }
}
```

**Visual states:**

| State | Visual |
|-------|--------|
| Idle | 56dp red circle (`#FFE53935`) with 2dp white border. No inner icon. |
| Recording | 56dp red circle (`#FFE53935`) with 2dp white border. White 20dp rounded-square (`RoundedCornerShape(4.dp)`) centered inside. |
| Stopping (transitional) | Same as Recording visual until Finalize fires; `enabled = false` during Stopping to prevent double-tap. |
| Disabled (during Stopping) | Red circle with stop square + `alpha = 0.6f` (brief Stopping state; transitions to Idle in <500ms typically). |

**State transition animation:**
- Idle → Recording: `AnimatedContent(targetState = isRecording)` with `ContentTransform(fadeIn(tween(200, easing = FastOutSlowInEasing)), fadeOut(tween(200, easing = FastOutSlowInEasing)))` for the inner content (stop square appears/disappears). Outer red circle and white border remain static — no animation on container shape.
- Duration: 200ms. Easing: `FastOutSlowInEasing`.

**Haptic:** `HapticFeedbackConstants.VIRTUAL_KEY` on tap — lightweight tick (same as filter selection, not shutter's LONG_PRESS).

**Touch target:** 56dp — exceeds 44dp minimum. No additional padding needed.

**Guard:** `isRecording` guard in ViewModel `onRecordTapped()` prevents concurrent recordings (D-26 pattern — matches `isCapturing` guard on shutter).

### 2. Recording Indicator (D-08)

**Position:** `Alignment.TopCenter` inside camera preview `Box`, `padding(top = 24.dp)`. Hidden when `recordingState is RecordingState.Idle`.

**Visibility:** `AnimatedVisibility(visible = isRecording, enter = fadeIn(tween(300)), exit = fadeOut(tween(300)))`.

**Layout:**
```
Surface(
    color = Color(0xCC000000),   // 80% black backdrop for legibility on any scene
    shape = RoundedCornerShape(16.dp),
    tonalElevation = 2.dp,
    modifier = Modifier.padding(horizontal = 12.dp, vertical = 4.dp)
) {
    Row(
        verticalAlignment = Alignment.CenterVertically,
        modifier = Modifier.padding(horizontal = 8.dp, vertical = 4.dp)
    ) {
        // Red dot
        Box(
            modifier = Modifier
                .size(10.dp)     // 10dp rendered dot inside 16dp touch-safe space
                .clip(CircleShape)
                .background(Color(0xFFE53935).copy(alpha = blinkAlpha))
        )
        Spacer(modifier = Modifier.width(8.dp))   // sm token
        // Timer text
        Text(
            text = elapsedFormatted,   // "MM:SS" from RecordingState.Active.elapsedMs
            style = TextStyle(
                fontSize = 16.sp,
                fontWeight = FontWeight.Medium,
                color = Color.White
            )
        )
    }
}
```

**Red dot blink animation:**
- `val blinkAlpha by animateFloatAsState(targetValue = if (blinkOn) 1.0f else 0.5f, animationSpec = tween(durationMillis = 500, easing = LinearEasing), label = "recordBlink")`
- `blinkOn` toggles via `LaunchedEffect(isRecording) { while (isRecording) { delay(500); blinkOn = !blinkOn } }` — 1Hz cycle (500ms on, 500ms off = 1 full cycle per second).
- Easing: `LinearEasing` — the blink pulse is a utility signal, not a choreographic motion. Linear gives an even flash characteristic matching camera convention.
- Alpha range: 1.0f (fully opaque red) → 0.5f (half-transparent red). Does NOT go to 0f — dot remains visible at half intensity so position is always apparent.

**Timer:**
- Format: `String.format("%02d:%02d", elapsedMs / 60000, (elapsedMs % 60000) / 1000)` — produces "00:00" through "01:00".
- Source: `RecordingState.Active.elapsedMs` incremented by `LaunchedEffect(isRecording)` → `delay(1000)` loop in ViewModel.
- Ticks every 1 second. Does NOT advance during `RecordingState.Stopping`.

**Backdrop:** `Color(0xCC000000)` — 80% opaque black pill behind dot + timer. Ensures readability on any camera scene (white backgrounds, bright outdoors). Uses `Surface` with `RoundedCornerShape(16.dp)` for pill shape — NOT a full-width bar.

**Semantics:** `Modifier.semantics { liveRegion = LiveRegionMode.Polite; contentDescription = "Recording: $elapsedFormatted" }` on the Row — announced to TalkBack every time timer updates (Polite = waits for current speech to finish).

### 3. Exit-During-Record AlertDialog (D-10)

**Trigger:** `BackHandler(enabled = isRecording)` intercepts system back press → sets `showDiscardDialog = true` in UI state.

**Implementation:** Material3 `AlertDialog` composable.

```kotlin
if (showDiscardDialog) {
    AlertDialog(
        onDismissRequest = { showDiscardDialog = false },  // same as Cancel
        title = {
            Text(
                text = "Recording in progress",
                style = TextStyle(fontSize = 16.sp, fontWeight = FontWeight.Medium)  // Phase 5: 2-weight system override (was titleMedium SemiBold)
            )
        },
        text = {
            Text(
                text = "Are you sure you want to discard this recording?",
                style = MaterialTheme.typography.bodyMedium
            )
        },
        confirmButton = {
            TextButton(
                onClick = { showDiscardDialog = false }   // Cancel — resume recording, no state change
            ) {
                Text("Cancel")
            }
        },
        dismissButton = {
            TextButton(
                onClick = {
                    showDiscardDialog = false
                    onDiscardRecording()   // → ViewModel.onDiscardRecording() → recording.stop() + delete pending URI
                },
                colors = ButtonDefaults.textButtonColors(
                    contentColor = MaterialTheme.colorScheme.error   // #FFB00020 destructive red
                )
            ) {
                Text("Discard")
            }
        }
    )
}
```

**Button placement:** Material3 `AlertDialog` places `confirmButton` on the RIGHT and `dismissButton` on the LEFT. "Cancel" (confirm = resumes/no-action) is RIGHT; "Discard" (dismiss = destructive) is LEFT. This is intentional: "Cancel" as the safe/right action follows Material3 convention where the affirmative/safe action is rightmost.

**Typography in dialog:**
- Title "Recording in progress": Explicit `TextStyle(fontSize = 16.sp, fontWeight = FontWeight.Medium)` — 16sp / Medium (500). Phase 5 reduces 3-weight system to 2; SemiBold eliminated. Override Material3 default `titleMedium` (which would be SemiBold).
- Body "Are you sure you want to discard this recording?": `MaterialTheme.typography.bodyMedium` — 14sp / Normal (400). Applied automatically by Material3 AlertDialog text slot.
- "Cancel" button: `MaterialTheme.typography.labelLarge` — 14sp / Medium (500). Material3 TextButton default.
- "Discard" button: `MaterialTheme.typography.labelLarge` — 14sp / Medium (500), tinted `colorScheme.error` (`#FFB00020`).

**Enter/exit motion:** Material3 default `AlertDialog` enter/exit animation — 200ms scale-from-center + fade. No override.

**onDismissRequest behavior:** Tapping outside the dialog (scrim) = same as "Cancel" — dismisses dialog, recording continues.

**Focus traversal in dialog:** Material3 handles focus trap inside AlertDialog automatically. Order: Title → Body → Discard (left button) → Cancel (right button).

### 4. Lock-During-Record Treatment (D-11, D-23)

**Principle:** Visual alpha signals locked state; no tooltip, no icon change, no overlay banner.

**Picker LazyRow items:**
```kotlin
FilterThumbnail(
    modifier = Modifier.alpha(if (isRecording) 0.5f else 1.0f),
    onClick = {
        if (isRecording) return@FilterThumbnail   // early return — tap ignored
        onFilterSelected(filter.id)
    }
)
```
- Alpha: 0.5f when recording (half-opacity — still shows which filter is active, but visually dimmed to signal non-interactivity).
- Haptic: NOT fired when tap is early-returned.
- Selected thumbnail border ring: remains visible at 50% alpha — selected state is preserved and communicated even when locked.

**Flip button:**
```kotlin
OutlinedButton(
    enabled = !isRecording,   // Material3 disabled state: 38% alpha text, 12% alpha container
    onClick = onFlipCamera
) {
    Text("Flip")
}
```
- Uses Material3 `enabled = false` — do NOT add manual `Modifier.alpha` on top (would double-dim beyond Material3 convention).

**Sticker gestures (InsectFilterScreen):**
```kotlin
Modifier.pointerInput(isRecording) {
    if (isRecording) return@pointerInput   // gesture lock — no detectTransformGestures attached
    detectTransformGestures { centroid, pan, zoom, rotation ->
        onStickerGesture(pan, zoom, rotation)
    }
}
```
- `pointerInput(isRecording)` key causes the block to restart when `isRecording` changes — correctly detaches/reattaches gesture detection.
- When locked: touch events pass through to the preview surface (no gesture consumed).

**Shutter button:** NOT locked. Photo capture during video recording is supported (CameraX 1.6 concurrent ImageCapture + VideoCapture). Shutter remains fully enabled with unchanged appearance.

### 6. Sticker Visual (D-03, D-05, D-06)

**Rendering layer:** Sticker is drawn via `StickerRenderer.onDraw(canvas, frame)` inside `OverlayEffectBuilder.setOnDrawListener`. This is NOT a Compose composable overlay — it lives in the same Canvas draw call as the filter sprites.

**Why OverlayEffect (not Compose):** Sticker must be baked into video and photo output streams. A Compose overlay would only appear on the preview surface. OverlayEffect routes through all three targets (PREVIEW, IMAGE_CAPTURE, VIDEO_CAPTURE).

**StickerRenderer draw logic:**
```kotlin
// canvas has already had frame.sensorToBufferTransform applied
canvas.save()
canvas.translate(state.offset.x, state.offset.y)
canvas.rotate(state.rotation, 0f, 0f)    // rotate around translated center
canvas.scale(state.scale, state.scale)
canvas.drawBitmap(
    activeFilterFrames[currentFrameIndex],
    -bitmap.width / 2f,   // center bitmap on translated origin
    -bitmap.height / 2f,
    paint
)
canvas.restore()
```

**Visual appearance:** Clean sprite render. No border ring. No drop shadow. No debug bounding box. No face-landmark dots (face detection disabled in this mode).

**Initial state (D-02):** When InsectFilterScreen composable enters composition, ViewModel sets `StickerState(offset = Offset(previewWidth / 2f, previewHeight / 2f), scale = 1.0f, rotation = 0f)`. Preview dimensions obtained from `OverlayEffect.Frame.size` on first draw callback and stored in ViewModel.

**Scale constraints (D-03):** `scale = (state.scale * zoomDelta).coerceIn(0.3f, 3.0f)`.

**Offset constraints (D-03):** Sticker center clamped to keep within preview bounds expanded by 50% of sprite dimensions in each direction — allows off-edge placement up to half the sprite width/height:
```kotlin
val halfW = (bitmap.width * state.scale) / 2f
val halfH = (bitmap.height * state.scale) / 2f
val clampedX = (state.offset.x + pan.x).coerceIn(-halfW, previewWidth + halfW)
val clampedY = (state.offset.y + pan.y).coerceIn(-halfH, previewHeight + halfH)
```

**Rotation:** Unbounded. `rotation = (state.rotation + rotationDelta) % 360f`.

**Sprite animation:** Same frame-sequence mechanism as FaceFilter mode (frame index driven by `Choreographer` or `OverlayEffect.Frame.frameTimeNanos` delta). Frame rate: `FilterDefinition.frameDurationMs` per sprite.

**Active filter sprite source:** `StickerRenderer.setActiveFilter(FilterDefinition)` loads frames from `AssetLoader` (same `preload(assetDir)` call as Phase 4). Initial filter on InsectFilterScreen entry: last-used filter restored from DataStore (CAT-05 pattern). If no saved preference: first entry in `FilterCatalog.all`.

### 7. Sticker Gesture Interaction

**Gesture handler placement:** `Modifier.pointerInput(isRecording)` applied to a full-screen transparent `Box` overlay sitting above `CameraXViewfinder` but below the control buttons (picker, shutter, record, flip). The control buttons intercept their own touch events first; any touch that lands on the preview area outside buttons is handled by the gesture modifier.

**Gesture API:** `detectTransformGestures { centroid, pan, zoom, rotation -> ... }` — single Compose API for combined pan + pinch-to-zoom + two-finger rotation.

**When `isRecording == true`:** `return@pointerInput` before `detectTransformGestures` — no gesture is consumed, no state mutation. Touch events fall through to viewfinder (no-op visually, prevents accidental sticker movement during recording).

**When `isRecording == false`:** All three gesture components (pan, zoom, rotation) processed simultaneously. Each delta accumulated into `StickerState` via `onStickerGesture(pan, zoom, rotation)` in ViewModel.

**Haptic feedback:** NOT fired during gesture moves (would be disruptive during continuous multi-touch). Fire `HapticFeedbackConstants.VIRTUAL_KEY` only when sticker reaches scale clamp boundary (0.3f or 3.0f) — single tap haptic indicating hard limit. This is Claude's discretion per 05-CONTEXT.md discretion list.

**Single vs multi-finger:** `detectTransformGestures` handles 1-finger drag (pan only), 2-finger spread (zoom + optional pan), 2-finger twist (rotation + optional pan). All via single API — no separate gesture detectors needed.

**No gesture start/end callbacks:** `StickerState` updates continuously as fingers move. No commit/cancel UX — any gesture change is immediately applied and persists in ViewModel StateFlow.

## Motion Specs

**INHERITED from 04-UI-SPEC, plus Phase 5 additions.**

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Filter thumbnail scale: unselected → selected | 200ms | `FastOutSlowInEasing` | User taps thumbnail (inherited) |
| Filter thumbnail scale: selected → unselected | 200ms | `FastOutSlowInEasing` | Different thumbnail selected (inherited) |
| Capture flash fade-in | 75ms | `LinearEasing` | Shutter tapped (inherited) |
| Capture flash fade-out | 150ms | `LinearEasing` | 225ms after shutter tap (inherited) |
| Record button idle→recording content swap | 200ms | `FastOutSlowInEasing` | Record tapped; `AnimatedContent` fade |
| Record button recording→idle content swap | 200ms | `FastOutSlowInEasing` | Stop tapped or auto-stop; `AnimatedContent` fade |
| Recording indicator appear (fadeIn) | 300ms | `LinearEasing` | `isRecording` becomes true |
| Recording indicator disappear (fadeOut) | 300ms | `LinearEasing` | `isRecording` becomes false |

## Implementation Notes for Executor

1. **Record button vs Shutter z-order:** Both are in the camera preview `Box`. Shutter is `Alignment.BottomCenter`; Record is `Alignment.BottomStart`. They share the same `bottom = 24dp` padding but are on opposite sides of the screen — no overlap possible in portrait. No z-order conflict.

2. **InsectFilterScreen reuses CameraScreen components:** The picker `LazyRow`, shutter button, flip button, and capture flash overlay are identical Compose components. Extract these into shared composables (e.g., `CameraPickerStrip`, `CameraShutterButton`, `CameraFlipButton`) in a `ui/camera/components/` package if not already done in Phase 4. Prevents code duplication between `CameraScreen.kt` and `InsectFilterScreen.kt`.

3. **Sticker Canvas coordinate space:** `StickerRenderer.onDraw()` is called inside `OverlayEffectBuilder.setOnDrawListener` AFTER `canvas.setMatrix(frame.sensorToBufferTransform)` is applied. The `StickerState.offset` must be in sensor-buffer coordinate space, not screen/dp space. Convert from preview-size coordinates: when user drags, the touch offset (in dp) must be scaled to match the OverlayEffect canvas coordinate space. Use `OverlayEffect.Frame.size` (available in draw callback) to compute scale factor. Store this scale factor in ViewModel for gesture-to-canvas coordinate conversion.

4. **BackHandler ordering:** `BackHandler(enabled = isRecording)` in `InsectFilterScreen` intercepts before navigation `BackHandler`. Ensure it is declared before the navigation graph's implicit back behavior — place it early in the composable body (before any other `LaunchedEffect` or `DisposableEffect` blocks).

5. **RecordingState.Stopping guard:** When `recording.stop()` is called, there is a brief period before `VideoRecordEvent.Finalize` fires. During this `Stopping` state, the Record button must be `enabled = false` to prevent double-tap. Also emit no `Toast` during Stopping — wait for Finalize to determine success or error.

6. **Timer LaunchedEffect leak guard:** `LaunchedEffect(isRecording)` with `while (isRecording) { delay(1000); elapsedMs += 1000 }` — the loop exits when `isRecording` becomes false because `LaunchedEffect` key change cancels the previous coroutine. No explicit cancellation token needed. Reset `elapsedMs = 0` when `RecordingState` transitions back to Idle.

7. **OverlayEffectBuilder branching for InsectFilter mode:** In `OverlayEffectBuilder.setOnDrawListener`, the draw callback receives `cameraMode` from the outer ViewModel scope. InsectFilter path calls `stickerRenderer.onDraw(canvas, frame)` only; FaceFilter path calls `filterEngine.onDraw(canvas, frame, faces)` only. Both paths call `debugOverlayRenderer.draw(...)` after if `BuildConfig.DEBUG && faces.isNotEmpty()` (faces will be empty in InsectFilter mode — effectively a no-op draw in InsectFilter).

8. **Picker strip identical to Phase 4:** InsectFilterScreen picker strip is 100% the same as CameraScreen picker. `FilterCatalog.all` (same 15 filters), same 72dp thumbnails, same selection animation, same DataStore persistence. The only visual context difference: picker appears on a screen with a free-placement sticker instead of a face-tracked filter. No picker spec change needed.

9. **HomeScreen Insect Filter button enabled state:** Phase 5 activates `InsectFilterScreen`. The HomeScreen `OutlinedButton("Insect Filter", enabled = false)` from Phase 4 must be changed to `enabled = true` and `onClick = { navController.navigate(CameraRoute(CameraMode.InsectFilter)) }`. Remove the Phase 4 "Coming soon" subtitle if added.
