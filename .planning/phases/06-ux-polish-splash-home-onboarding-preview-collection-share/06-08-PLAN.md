# 06-HANDOFF.md (Phase 6 Device Runbook for Xiaomi 13T)

## Pre-flight
- Build: clean APK installed via `./gradlew :app:installDebug` or `adb install -r app/build/outputs/apk/debug/app-debug.apk`
- Device: Xiaomi 13T (Android 13+, HyperOS), USB debugging on, `adb shell svc power stayon usb` set
- Pre-clean DataStore (for first-launch test): `adb shell pm clear com.bugzz.filter.camera`

## Hard Gates (must PASS — 13 items mapping to Phase 6 requirements)

### UX-01 — Splash
- [ ] Fresh-clear (cleared prefs above) → launch app
- [ ] Splash shows Lottie + "Bugzz" brand text
- [ ] Auto-advances within ~1.5s (≤2s acceptance window)

### UX-02 — Onboarding (first launch)
- [ ] Splash → 3-page Onboarding pager appears
- [ ] Swipe through 3 pages — title + body strings match UI-SPEC §3
- [ ] Skip on page 1 → Home (subsequent launch confirms onboarding_completed=true)
- [ ] Force-stop + relaunch → Splash → Home (no onboarding) — confirms D-07 / UX-09 success criterion #5

### UX-03 — HomeScreen polish
- [ ] Settings gear (top-right) → SettingsScreen opens
- [ ] My Collection button → CollectionScreen opens
- [ ] Face Filter / Insect Filter → respective Camera mode (Phase 4/5 carry — should still work)

### UX-04 — Preview after capture
- [ ] Capture photo (Phase 3 flow) → PreviewScreen with Coil AsyncImage
- [ ] Capture video (Phase 5 flow) → PreviewScreen with ExoPlayer auto-play loop
- [ ] Action bar: Done + Share + Delete + Retake icons + 10sp labels visible
- [ ] Done → returns to Camera (file remains saved — verify via Files app)
- [ ] Retake → returns to Camera

### UX-05 — Collection grid
- [ ] CollectionScreen shows 3-column grid of all DCIM/Bugzz/* artifacts
- [ ] Photos render via Coil; videos render via MediaMetadataRetriever thumbnail + center play overlay
- [ ] Sort order: newest first

### UX-06 — Collection → Preview navigation
- [ ] Tap thumbnail → PreviewScreen opens with that URI
- [ ] Retake from Preview entered via Collection → returns to Collection (not Camera)

### UX-07 — Empty state
- [ ] Clear app data again (or wait until empty) → Open Collection → EmptyStateColumn renders
- [ ] Lottie + "No bugs captured yet" + "Open Camera" button visible
- [ ] Tap "Open Camera" → HomeScreen

### UX-08 — Delete confirmation
- [ ] Preview → Delete → AlertDialog "Delete this artifact?" with Cancel + Delete buttons
- [ ] Cancel → dialog dismissed, artifact remains
- [ ] Delete (red text) → artifact deleted from MediaStore (verify in Files app); pop back to Camera or Collection per source

### UX-09 — Settings stub
- [ ] Home → Settings gear → SettingsScreen TopAppBar "Settings" + 4 rows
- [ ] Version row shows BuildConfig.VERSION_NAME
- [ ] Privacy Policy tap → Toast "Coming in next release"
- [ ] Rate the App tap → Toast "Coming when published to Play Store"
- [ ] About row read-only with trailing "Bugzz — Bug filter prank camera"
- [ ] Back arrow returns to Home

### SHR-01..04 — Share
- [ ] Preview → Share → Android share sheet "Share via" appears
- [ ] Share sheet shows installed targets (WhatsApp, Messenger, Zalo if installed on user's device)
- [ ] Pick a target → receiving app shows the photo/video with overlay intact (SHR-04)
