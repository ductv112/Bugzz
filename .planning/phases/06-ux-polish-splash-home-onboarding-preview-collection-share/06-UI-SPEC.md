### 1. LottiePlayer — Shared Wrapper (D-26)

**Package:** `ui/components/LottiePlayer.kt`

**Purpose:** Reusable wrapper around `lottie-compose` `LottieAnimation` composable. Provides default looping config. Used by SplashScreen, OnboardingScreen, EmptyStateColumn.

**Lottie asset strategy (D-29 choice):** Copy `home_lottie.json` from `reference/raw_extract/res/raw/home_lottie.json` to `app/src/main/assets/lottie/home_lottie.json`. Use `LottieCompositionSpec.Asset("lottie/home_lottie.json")`. This is simpler than a PNG-flipbook renderer and produces the intended animated UX (per RESEARCH.md primary recommendation).

**Signature:**
```kotlin
@Composable
fun LottiePlayer(
    assetPath: String,               // e.g. "lottie/home_lottie.json"
    modifier: Modifier = Modifier,
    iterations: Int = LottieConstants.IterateForever,
    isPlaying: Boolean = true,
)
```

**Internal behavior:**
- `rememberLottieComposition(LottieCompositionSpec.Asset(assetPath))` — loads JSON from assets
- `animateLottieCompositionAsState(composition, isPlaying, iterations)` — drives progress
- `LottieAnimation(composition, progress = { progress }, modifier)` — renders

**Defaults:**
- `iterations = LottieConstants.IterateForever` — loops indefinitely (EmptyStateColumn usage)
- Override with `iterations = 1` for Splash (single play)

```
Column(
    horizontalAlignment = Alignment.CenterHorizontally,
    modifier = Modifier.align(Alignment.BottomCenter).padding(bottom = 48.dp)
) {
    // Page indicator dots
    Row(horizontalArrangement = Arrangement.spacedBy(8.dp)) {
        repeat(3) { index ->
            val isActive = pagerState.currentPage == index
            Box(
                modifier = Modifier
                    .size(if (isActive) 12.dp else 8.dp)
                    .clip(CircleShape)
                    .background(
                        if (isActive) Color(0xFFE53935) else Color(0xFF9E9E9E)
                    )
            )
        }
    }
    Spacer(Modifier.height(24.dp))   // lg token
    Button(
        onClick = {
            if (pagerState.currentPage == 2) onComplete()
            else scope.launch { pagerState.animateScrollToPage(pagerState.currentPage + 1) }
        }
    ) {
        Text(
            text = if (pagerState.currentPage == 2) "Get Started" else "Next",

        )
    }

```

**Accessibility:**
- Skip button: `contentDescription = "Skip onboarding"`, `Role.Button`
- Next button: contentDescription = `"Next page"` on pages 0–1, `"Get started"` on page 2, `Role.Button`
- Page indicator row: `Modifier.semantics { contentDescription = "Page ${pagerState.currentPage + 1} of 3" }` on the Row — announced on page change
- HorizontalPager swipe accessible via `semantics { horizontalScrollAxisRange = ... }` — Compose Foundation HorizontalPager handles this automatically

## Implementation Notes for Executor

1. **Lottie asset path:** Use `LottieCompositionSpec.Asset("lottie/home_lottie.json")` — reads from `assets/lottie/`. If referencing `res/raw/`, switch to `LottieCompositionSpec.RawRes(R.raw.home_lottie)`. Do NOT mix the two patterns.

2. **Routes.kt must change BEFORE BugzzApp.kt:** `PreviewRoute` changes from `data object` to `data class(val uri: String)`. If BugzzApp.kt is compiled first with the old `data object` route, the build fails with a serialization error. Update Routes.kt in the same commit or earlier.

3. **ExoPlayer release is mandatory:** Always pair `remember { ExoPlayer.Builder(context).build() }` with `DisposableEffect(exoPlayer) { onDispose { exoPlayer.release() } }`. Missing `release()` causes AudioFocus + SurfaceTexture leaks caught by LeakCanary (T-06-03 threat).

4. **MediaStore delete on IO dispatcher:** `MediaStore.delete(uri, null, null)` or `context.contentResolver.delete(uri, null, null)` must run on `Dispatchers.IO` from ViewModel scope. Never call from the main thread.

5. **CollectionRepository URI construction:** Use `MediaStore.Images.Media.EXTERNAL_CONTENT_URI` for photo rows and `MediaStore.Video.Media.EXTERNAL_CONTENT_URI` for video rows when constructing `ContentUris.withAppendedId(...)` — NOT the generic Files URI (some apps reject it). The `_ID` from a Files-table query is the same as the Images/Video table ID.

6. **Onboarding dot size vs scale:** Use `size(if (isActive) 12.dp else 8.dp)` — explicit size, not `graphicsLayer { scale }`. `graphicsLayer` scale causes layout jank on small elements because the composable still occupies its pre-scale layout size. Direct size change with `animateIntAsState` for smooth transition is optional; static size change is sufficient for Phase 6.

7. **DeleteConfirmDialog `confirmButton` / `dismissButton` convention:** In Material3 `AlertDialog`, `confirmButton` renders on the RIGHT and `dismissButton` on the LEFT. To match Phase 5 convention (Cancel = right = safe action, Delete = left = destructive), pass Cancel as `confirmButton` and Delete as `dismissButton`. This is counterintuitive naming but correct visually.

8. **VideoPreview `remember(uri)` key:** `remember(uri)` ensures a new ExoPlayer is created if URI changes. Without the key, navigating between two video items in Collection would play the original item's video.

9. **TopAppBar title typography:** `SmallTopAppBar(title = { Text("Settings") })` — Material3 manages `titleMedium` TextStyle for the title slot. This is system-applied, not a custom `Text` call with SemiBold. Phase 6 does NOT add any custom Text override on TopAppBar titles; the system-managed style is acceptable and not in conflict with the 2-weight rule (which governs explicit `Text(style=...)` calls only).

10. **Phase 3+4+5 fix preservation:** Before touching `CameraViewModel`, `InsectFilterViewModel`, or `OverlayEffect` code, verify that these fixes remain intact:
    - `isCapturing` guard (Phase 3 dafc21e)
    - `bindJob.cancel()` (Phase 3 9abbd0b)
    - `captureFlash` inside onSuccess (Phase 3 4e94591)
    - `assetLoader.preload(def.assetDir)` (Phase 4 514410c)
    - `isRecording` guard (Phase 5 D-26)
    - `cameraMode = CameraMode.InsectFilter` propagation (Phase 5 gap-01)
    - `StickerRenderer.setPreviewSize` + matrix reset (Phase 5 gap-02)
